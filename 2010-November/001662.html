On Tuesday, 2. November 2010 17.45:27 MSEide MSEgui wrote:
> >> IMHO, relative path should change to application path (where
> >> executable file is stored).
> >
> > "./" means current directory (cd command). The directory of the exe is
> > often write protected.
> > _______________________________________________
>
> How to adjust path to application path?

On Linux "~/" and "^/" return the <HOME> directory, on Windows "~/" returns 
the result of SHGetFolderPathW(CSIDL_PROFILE), "^/" the result of 
SHGetFolderPathW(CSIDL_APPDATA).
The applicationpath will be returned by sys_getapplicationpath() (unit 
msesysintf).
