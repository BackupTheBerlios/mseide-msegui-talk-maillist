> I don't think so. I never used onkeydown/up on edit widgets. There probably
> are better solutions. Do you have an example where onkey* is necessary for

I use onkeydown/up event in POS (Point Of Sale) app. I use
tdbdropdownlisteditdb (wsn) to show serial number list of products. If
user select a serial number and press enter, he asked "Add another
serial number?", if user answer "Yes", cursor focused to "wsn", but if
user answer "No", cursor focused to product code. Unfortunately, if
cursor focused to product code, enter from showmessage dialog is sent
to product code.

For me, onkey* event on edit widget is important to create app that
prepared to quick operation app. Please you don't delete this event,
this is standard event for all GUI library.
