I suggest to run the crashing MSEide in gdb in order to get more info.
The most convenient way is to use the latest working MSEide you have (MSEide 
2.4.2 for example) and to do 'Project'-'Open'-apps/ide/mseide.prj. 
Then 'Project'-'Save as'-mseide_md.prj in order to get a copy of the project 
file which is independent of the SVN checkout. Update the settings 
of 'Project'.'Options'-Macros' and 'Make' if necessary and compile and run 
the loaded project.
