On Friday, 26. November 2010 15.26:42 Christian Nobel wrote:
> This can be done in numerous ways, but as I understand the
> tdbdropdownlisteditdb could be a way.
>
> However I miss some light to understand the zillion of parameters for
> the tdbdropdownlisteditdb.
>
If the key field is 32bit numeric use tdbenumeditdb, if it is 64bit 
tdbenum64editdb, if the keyfield is a string use tdbkeystringeditdb.
For big lookup datasets where the performace does not suffice use a 
tdblookupbuffer and tdb*editlb.

> As I understand it is possible to have two (or more) columns in a
> dropdown(edit), but can I chose to decide which column is going to be
> stored in the table, and another displayed (so that it is the
> customername that is displayed, but custid selected)?
>
dropdown.valuecol defines which column will be displayed in editwidget.

> Is it also possible to extend the dropdown to more columns, so the value
> from the fist column is stored in one field, and the value form the
> second in another field, and so fort?

Extending is possible by seeting dropdown.cols.col > 1. Data storing in 
additional fields must be done in code. It is probably better to use a 
master-detail relationship because of data duplication.
