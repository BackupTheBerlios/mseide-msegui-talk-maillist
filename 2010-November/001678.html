First, I think users should read README.TXT. There are not many steps to do 
for installing MSEide+MSEgui so I think we can expect that users make the 
necessary settings. I see from the download statistics that often the Windows 
binary package is the solely downloaded file, MSEide+MSEgui is useless 
without the source package...
What also sometimes happens is that users read 'Project'-'Open' 
as 'File'-'Open'.

Because ${MSEDIR} is located in settings dialog it is always defined, an empty 
string as macro value is sometimes needed too, so I think there is no general 
solution. Unresolved macro references are not touched so they show up in the 
popup hint of the option edits. Please report if you find an edit widget in 
project options where the macro expansion popup hint is not enabled.

> Maybe show an error state like the "shortcut conflict" in the Shortcuts
> dialog. The field is highlighted in error if the macro doesn't exist or is
> empty, but doesn't force the user to take action immediately. At least the
> user is notified though.
>
Complicate to implement. Don't you think the popup hints are enough? It 
wouldn't help for empty ${MSEDIR} anyway if users don't 
check 'Project'-'Options'.
