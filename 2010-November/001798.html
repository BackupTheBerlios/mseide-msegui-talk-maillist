I want to add that I used Turbopascal/Delphi many years starting with TP4
IIRC.
The current design of MSEgui is my fourth approach of a GUI framework. The
first was based on Delphi1, then I had to rewrite much code because of
Delphi 32 bit, so I started again with using VCL based components. At
appearance of Kylix I wanted to go cross platform so I started again from
scratch with CLX. You probably know what happened to Kylix.
After that experience I came to the conclusion that the software industry is
not able to deliver the necessary software development tools because of the
short lifecycles and the profit only driven decisions.
So I thought somebody must do it, went opensource and started from scratch
again with a new design avoiding some flaws of the VCL/CLX based
predecessors...
MSEide+MSEgui with exception of the DB-components still compiles with
Delphi7 BTW.
