It is a verry different approach. MSEifi has the MSEgui environment on 
clientside, the connection between client and server transports *.mfm, data 
and events only. gtk3 rendering by html5 has the GTK environment on server 
side, the client is a display and mouse/keyboard frontend only AFAIK.
It looks about the same as an X-remote session to me. Instead an X-server it 
uses Firefox and html5.
