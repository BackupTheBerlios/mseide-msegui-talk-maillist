Graeme Geldenhuys wrote:

> 
> But how can I have help for a whole Window (eg: Configure MSEide dialog)
> and not specific to a component on that window. I can't set focus to the
> Form itself. So if the OK button has focus and I press F1, the OK button
> is passed to the help handlers. Now if the OK button doesn't have any
> helpcontext information it stops there. But how can I tell MSEgui to
> ripple the help request through the component hierarchy up to the "top
> level" form. eg: OK button has no helpcontext, try the button's parent, so
> the Form itself is tried. Ah, the Configure MSEide dialog has helpcontext
> info, now launch DocView with that information.
> 
> Any hints?
> 
twidget.helpcontext property returns the first found value searching the
hierarchy upwards. Other sources for helpcontext values are
application.activehelpcontext and application.mousehelpcontext.
