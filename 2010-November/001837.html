> This looks interesting, where can I learn more?

The tifi*link components can be used as connectors between gui and business
logic.

If MSEide is compiled with -dmse_with_ifi_rem there are additional
experimental components for remote or local client-server connections. The
idea is that MSEgui form resource data (*.mfm equivalent) is transferred
over data streams (pipes, sockets...) from server to a client "MSEgui
interpreter". The client instantiates the forms/modules. The ifi components
on the client form/module instances communicate with the server application
by the stream connection and exchanges data and events. An example is here:
http://svn.berlios.de/viewvc/msedocumenting/mse/trunk/help/tutorials/mseifi/ifipipedemo/

If MSEide is compiled with -dmse_with_pascalscript there are additional
experimental components in tab 'PaSc'. They can be used with
a 'File'-'New'-'Form'-'Scriptform' in order to execute pascalscript code on
client. There are PascalScript import units and components for basic
testing purpose only at the moment.
