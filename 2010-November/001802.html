Sorry it butt in like this, and no I didn't check in MSEgui so maybe
this message can be ignore. But doesn't MSEgui have a distinction
between the various keyboard related events. eg: in fpGUI I have
OnKeyDown, OnKeyUp, OnKeyPress.  I would have recommended Wahono use
OnKeyPress event instead of the KeyUp/KeyDown events, if it was fpGUI,
because then he would not have that double key issue.

If you are not sure what's the difference between KeyUp/Down and
KeyPress. The later is the key that is generated from a single key
press like eg the letter a, or the key/symbol generated from a
combination of key presses (eg: AltGr+^+e will generate the ê letter
in the OnKeyPress event). In this last example, the OnKeyDown and
OnKeyUp will occur 3 times each as each keyboard key is pressed and
released, and the OnKeyPress will occur only once for the ê character.


So isn't Wahono simply using the wrong keyboard event for the job?
