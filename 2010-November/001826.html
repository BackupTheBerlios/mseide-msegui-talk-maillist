Op 2010-11-05 09:14, Martin Schreiber het geskryf:
> Trunk 4077 has 'Project'-'Options'-'Settings','Load' and 'Save'
> (experimental).
> 'Save' stores the values of 'Project'-'Options' in the 'Settings' *.prj
> file. 'Load' loads the values with exception of the 'Edit' values
> from 'Settings' *.prj file.

This sounds like a round-about way of doing it, but I'll give it a try this
week and see how it goes.


With these new changes, what exactly is considered "session information".

Considering multiple developers in a team, working on the same project, my
list would include:

 - IDE layout (panels, docking, position of windows, size of windows,
     position of splitter between docked windows etc)
 - Debug info: breakpoints, watches etc whatever a developer would have
     setup while they did there last debugging session.
 - Opened files, plus cursor locations etc.
 - Last used units (in Open dialog)

These are all things each developer would use and change, but doesn't
relate directly to a specific project, but rather to the developers working
style and preference.


How exactly are we supposed to use this new project setting?

 - When must we populate that new edit field
 - When does the IDE save session info
 - When does the IDE load session info
 - Why multiple .prj files for the same project
 - Which project (*.prj) are we supposed to open or save when
   go "Project > Open" or "Project > Save"

As you can guess, I'm still a bit confused about some things. :) Why can't
the IDE simply save and load session information automatically in the
~/.mseide/ directory when you open/save the official project .prj file? Why
pollute the project's source directory with files that must not be checked
into a code repository. Why have multiple .prj files at all?


By the way, keeping the session information out of the .prj file also has
the benefit that your Project Templates have "clean" templates, not with
some developer's specific file locations or what files they last had open
or their IDE window layout etc.
