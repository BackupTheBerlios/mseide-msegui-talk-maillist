On Friday, 5. November 2010 19.28:30 Sieghard wrote:
> Hallo Martin,
>
> Du schriebst am Fri, 05 Nov 2010 08:14:32 +0100:
> > There is no special session file, a 'Settings' file can be opened as
> > projectfile, a projectfile can be stored as 'Settings' file.
> >
> > Now, that is "orthogonal design", isn't it? ;-)
>
> Not at all, that's _parallel_ design - you can use somethng as something
> else, and vice versa.
> Orthogonal would mean you can use something disregarding of something
> else, because neither has any influence at all on the other.
> Mathematicians define a scalar product for this, which measures the
> "impact" of one "vector" on another, and define two "vectors" as being
> orthongonal if this "impact" vanishes.
> You see?

Yup, thats one of the design principles of MSEgui. The features should be as 
independent (orthogonal) as possible. In the example the storage of "project 
properties" and "session info" and there use are independent and can be 
freely combined in the same file or project instance.
You will find many such examples in the design of MSEgui. For example sets are 
preferred over enums. Such a design allows applications which were not 
foreseen at design time. The split of project options and session info was 
not foreseen at design time of the statfile environment. Nonetheless the 
implementation of the feature was possible with the existing environment.
