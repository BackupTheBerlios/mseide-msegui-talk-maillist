Graeme Geldenhuys wrote:

> 
> My issue now is that the --macrodef logic seems inconsistent. The problem
> being that the Project - Options - Macros take precedence, whereas
> initially I thought the command line parameter takes precedence (the
> newest definition of the macro).
> 
> I would have expected the latter to be the default. After all, that is how
> it works when overriding the global parameters. If no such macro exists,
> define one. If a global one exists, override it. I would have expected the
> same to be applied to the project macros. Just a thought.
> 
The precedence is:

1. Values stored in <HOME>/.mseide/mseideli.sta, editable
by 'Settings'-'Configure MSEide'. This file could be used for different
applications, the dialog called by 'Configure MSEide' is a separate module.

2. Evironment variables. Possible names:
FPCDIR, FPCLIBDIR, MSEDIR, MSELIBDIR, SYNTAXDEFDIR, TEMPLATEDIR,
COMPSTOREDIR, COMPILER, DEBUGGER, EXEEXT, TARGET.

3. Macros defined by --macrodef commandline parameter.

4. The settings in the *.prj file, editable by 'Project'-'Options'.

The last wins.
