On Sunday 07 November 2010 15:25:45 MSEide MSEgui wrote:
> If I press enter from message dialog (yes/no focused), enter key
> (key_return) is sent to other widget where showmessage is called.
>
> Please you place tstringedit, and create event on onkeyup event, than
> paste this code below :
>
> procedure tmainfo.tstringedit1_onkeyup(const sender: twidget;
>                var ainfo: keyeventinfoty);
> begin
>  if ainfo.key=key_return then begin
>   showmessage('Enter is pressed!');
>  end;
> end;
>
> After you run program, please press enter from tstringedit1, you will
> got message 'Enter is pressed!' twice.
>
Seems as designed to me. The edit has the focus, in onkeyup you call 
showmessage, in modal dialog with default button 'OK', you press Enter again, 
key_return keydown clicks the button and closes the message window, the edit 
widget has the focus again and receives the key_return keyup event. Press the 
OK button with mouse or Space key and there will be no second message window.

Do you remember, I did not want to add all this published event properties to 
the edit widgets because using them is not so easy as it seems? ;-)
What do you want to achieve?
