>
> I think this is about time to execute action in event handlers. I
> think we don't need onkeypress if execute default button doesn't
> affect other widgets (I think this is medium class of bug). Currently
> in editwidget, you can't call showmessage if user press enter to
> execute "OK" button, because enter from showmessage is sent to
> editwidget events.
>
You still did not answer my question why you need to handle onkeyup in an edit 
widget. The actions are done in dokeydown() and not in dokeyup().
End of editing is handled in onsetvalue and ondataentered. With checking 
onkeyup for enter key you will not get notified by end of editing because of 
focus exit or because of a menu or button press canclose() query.
If a keydown event changes the focused widget it is logical for me that any 
further keyevents (the keyup of enter key for example) go to the current 
focused widget.
