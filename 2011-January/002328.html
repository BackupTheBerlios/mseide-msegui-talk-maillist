Since in LINUX, "fixed" doesn't have an alias in "/etc/fonts/fonts.conf"  
thus manual editing of this file is necessary which means problems with  
automatic installation of MSEgui apps.

The current font situation for application addressing font generics is as  
follows:

LINUX & win-32 share 3 generic font sets (Linux - directly via FontConfig,  
win-32 - via font substitutions):

Courier   : LINUX="Nimbus Mono L" (PS-font) / Win-32="Courier New" (TTF)
Times     : LINUX="Nimbus Roman No9 L" (PS-font) / Win-32="Times New  
Roman" (TTF)
Helvetica : LINUX="Nimbus Sans L" (PS-font) / Win-32="Arial" (TTF)

For addressing non-generics (exact selecting "Times New Roman",..),  
there's no difference for LINUX & win-32.

PS:
Current LINUX distros are equipped with multiple charset ( incl  
iso10646-1, rechecked today with FontForge) Type1 fonts for the generics  
(and XOrg is capable of charset selection within them) thus the magic with  
TTF for these fonts isn't needed anymore.
