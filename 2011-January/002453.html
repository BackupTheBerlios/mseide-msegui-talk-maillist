On Fri, 2011-01-14 at 03:46 +0700, Sri Wahono wrote:
> Is git client support windows too?

Yes it supports just about any OS out there. Linux, BSD, Unix, Mac,
Windows, Haiku, and even a native Java implementation exists (used in
Eclipse project). And _please_ ignore all the OLD postings still
floating around on the internet that the Git support under Windows is
bad or slow. That was only so when the Git project just got started. Git
and Windows support is just as good as under Linux now.


>  But AFAIK, SVN client is more famous for user.

That depends on who you ask. :-)  There is a reason why so many high
profile (and large) projects like Qt, KDE etc all moved to git. No other
scm comes close. And just to be clear, git works just as well with small
one man projects. I even use a quick git repository when I work on a
small bug testing project or prototype idea. 'git init .' and you have a
repository to work with.

Git obviously comes with the command line client, but it also includes
as standard, two visual tools. 'git gui' which allows you pick which
line(s) in a file, or which hunks in a patch, or the whole file to
commit. Then there is also 'gitk' which is a visual repository viewer,
so you can visually see the different branches, how merges occurred
etc...

You DON'T need any of the other git GUI tools like gitg, kgit,
TortoiseGit etc.. They tend to be very slow, simply have too little git
features implemented, and it all just gives git a bad name. Rather use
the tools that are supplied with git and supports all the features git
has. They might not look pretty, but they are very functional and fast.
