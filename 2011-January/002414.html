Op 2011-01-11 21:50, Sieghard het geskryf:
> It might even happen that your installer cannot run because of a missing or
> mismatched library, even though the program to be installed _would_ run if
> simply copied to the system without the installer.

And that's the beauty of MSEgui and fpGUI toolkits - very low
dependencies, so something like an Installer can "just run". No JRE,
GTK1, GTK2 or Qt libs required - just the base X11 is needed. I tested
this theory with fpGUI's visual forms designer (a complex piece of code
using all available widgets). I ran it on Suse 9 without problems or any
installation - that's a Linux distro over 10 years old (a lifetime in
the Linux world).

That is why I think installers created with Java, GTK1, GTK2 or Qt is a
joke, because for any of those to run, you need a set of runtime
libraries. NOT so for MSEgui or fpGUI. So creating a Installer program
with the latter two is a perfectly sane idea.
