Debian package installers may be very powerfull ( user interaction,
stoping daemons then editing their configs then restarting,
dropping/creating users & databases, network activities, etc ) but aren't
GUI aware as to Open/Save/Create dialogues etc, AFAIK.
But it's easy to enforce only to install from X so that simple & small GUI
helper[s] integrated into installer would display all necessary dialogues
& return their results to the installer core.

As to different distros, there's a tool for repackaging between distros -
"alien".

My opinions about MATURE programs in Linux:

1) programms must only be installed by root into predefined catalogues
(/usr/lib/{app}, /usr/local[/lib]/{app})
2) programms must be versioned & integrated into distro packaging system
3) programms must declare preinstalation dependencies (in terms of distro
packaging) & exit installing if not met
4) programms must create upper-level soname-symlinks, all needed groups &
users etc, if needed.
