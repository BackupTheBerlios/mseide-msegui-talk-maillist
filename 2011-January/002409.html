For future projects you might consider using a Object Persistence
Framework like tiOPF [http://www.tiopf.com]. tiOPF supports about 9 or
more different database backends. You can switch between any of them at
compile time or runtime without changing a single line of code in your
application.

We use tiOPF in all our database and even non-database projects. I've
also implemented Model-GUI-Mediator (MGM) design pattern into tiOPF.
This allows you to use any standard (non-db) components and make them
object-aware. MGM also works with Delphi's VCL, Lazarus's LCL and fpGUI
Toolkit. It's really easy and very little code to add a new GUI toolkit
like MSEide to the list.

For more information see:

  tiOPF:   http://www.tiopf.com
  MGM:     http://opensoft.homeip.net:8080/articles/
