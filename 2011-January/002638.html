Well, Windows mainly ignores case anyway, so it doesn't really matter what
combination of case the name displays. The Explorer even "pretty cased" all
file names, i.e. has the abominous habit to capitalize every single "word"
of them, like is customarily done with song titles. (Like files were songs.)
(BTW, you could name the "Program Files" directory "PROGRAM FILES",
"program files", even "pROgRam fiLEs" or so, it would still be shown as
"Program Files" in the explorer... At least up to WXP, AFAIR)

But one _might_ be a rather ill advised to use a file name containing a
plus sign ("+") under windows, because this was used by "copy" as a file
concatenation operator for copy operations, and might still work that way
with command.com and cmd.exe.
