If a doc viewer were implemented, I would prefer DVI, PS or PDF to RTF
because RTF is MS proprietary software.

I prefer tar -xzvf software.tgz along with configure, make and make
install.  This packaging allows flexibility as well as the ability to
analyze the software before installing. If the end user cannot figure
it out then they should :
  1.) not have sufficient permissions on the system to install
executables, thereby avoiding pitfalls.
  2.) pay a professional, just as they would for any other service
that is beyond their skill level.

Thats my two cents worth (probably over valued),
Brian
