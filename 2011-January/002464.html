> and DB-aware components (duplicates of all non-db components).

Correct, although the MSEgui components are more sophisticated than there 
predecessors. Connectors between GUI and business logic exist too (ifilink 
components in tab 'IfI').
My experience is that the possibilities to make most efficient user interfaces 
were limited by tools which strictly use standard GUI widgets and no 
specialized components, code and settings in GUI part.
Very bad are web applications. While developing the remote MSEifi components I 
learned that is impossible to achieve the quality of a good desktop 
application with its specialized components and auxiliary code in the forms.
It may work for simple applications or not very discerning users but not for 
top applications.

> That is a ridiculous way of designing maintainable database aware apps.
> Yes it fits great with Delphi's "RAD development" sales pitch. But as
> far as I'm concerned, that is just ok for small apps or prototypes.
>
Disagreed. It is absolutely possible to make a clean and easy maintainable 
application structure with the RAD possibilities of MSEide+MSEgui.
