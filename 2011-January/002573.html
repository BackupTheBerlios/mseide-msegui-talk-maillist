> But on russian WINNT5, console encoding is cp1251 but GUI is uni-coded and
> cp1251 isn't the default layer in UNICODE so there're huge troubles with
> dipalying text constants (written in source code) in messageboxes & GUI

In MSEgui the source must be stored in utf8 and the compiler option -Fcutf8 
must be used.
Lazarus users can *not* use -Fcutf8 therfore the problems they have to work 
with unicode string constants. With -Fcutf8 FPC converts the string constants 
to utf-16 on compiletime if necessary.
