Op 2011-01-11 21:44, Sieghard het geskryf:
> need to "install" it - what the heck _does_ an installer do, anyway?
> Cannot a program just simply _work_?
> If the program needs any settings stored, it may do so at the usual places,
> but it should definitely _not_ need that to run correctly, and it should not
> save them without consent from the user ([ideally]).


Have you actually written a program that got deployed? Many apps these
days need a database server, the installer can even install such a
database server for them. Such database apps would require a database to
exist. The installer can create such a database. For the user to run the
app, the need to find it - the installer can create a icon for them in a
easy to find location.

The list goes on and on....

It's only very small utility/toy apps that just have a single executable
and "just works". Even MSEide (as Martin ships it) is pretty useless on
it's own. It needs the MSEgui source code or at least the compile units
of them, before MSEide can be used for anything. It needs project
templates to get a project started, it needs the FPC compiler etc... So
even Martin's "simple deployment idea" is flawed.
