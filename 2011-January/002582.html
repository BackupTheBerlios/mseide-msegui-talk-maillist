Without -Fcutf8 every string constant assignment to a UnicodeString or calling 
unicode Windows system functions (*W instead of *A) must be explicitely 
converted because the compiler does not know the encoding and assumes ISO 
8859-1.
"
 theuniucodevar:= UTF8decode(
        'theunicode constant in utf8 but the compiler assumes ISO 8859-1');
"
-> without -Fcutf8 working with UnicodeString is difficult. With -Fcutf8 the 
conversions source > unicode <> system encoding are done automatically by the 
compiler and the UnicodeStringManager.
