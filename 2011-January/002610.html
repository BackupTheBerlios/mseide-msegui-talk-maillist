On Thursday, 20. January 2011 20.34:06 Sieghard wrote:

> So the meaning of your statement seems to be that all file and path names
> in MSEGUI are stored as wide character strings (USC-2, AFAIR) and handed to
> the system as such, if applicable (for Linux, they would have to be
> converted to UTF-8).
> Is that right?

It will be converted to the current system encoding. On Linux today it is 
ususally utf-8. For Windows 
"
/C:/WINNT/system32/thefile.abc
"
will additionally be converted to
"
C:\WINNT\system32\thefile.abc
"
BTW the reason for " /C:/WINNT/system32/thefile.abc" is to easy determine if a 
path is a relative or a root path and to use the same paths on linux and 
Windows in a multi-boot environment where Windows partitions are mounted at 
drive character directories in Linux.
