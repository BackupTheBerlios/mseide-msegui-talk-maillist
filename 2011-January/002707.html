>> Linux?? I thin GTK use 72 PPI because AFAIK GTK use libcairo for drawing lib.
> I try with this command in Linux.
>
> xdpyinfo |grep dimensions


Do the following instead.

  $ xdpyinfo | grep inch


That will tell you what dpr x11 uses, but you can change KDE and Gnome
to use custom font dpi setting different to x11.
For GUI apps you need to use a layout manager, or calculate widget
size based on the ratio between what the form designer used, and what
the system is set at on which the app is running.  Form printers I
have no idea how tha works, but you can set the dpi of some printers
in the print dialog.
