> For  screen display :
> Windows  use 96 PPI (DPI), so PPMM = 3.78 or PPCM = 38 (rounded as in
> table). Macintosh use 72 PPI (DPI), so PPMM = 2.83 or PPCM = 28 (rounded
> too). Linux?? I thin GTK use 72 PPI because AFAIK GTK use libcairo for
> drawing lib. AFAIK, Graeme use 96 DPI fo his fpGUI.

Well, Linux is _meant_ to use what the user wants or, if applicable, the
monitor reports. (Otherwise, it seems to use a default value of 72 dpi or
so). E.g., on my system the utility "dpyinfo" reports (among others):

screen #0:
  dimensions:    1600x1200 pixels (534x401 millimeters)
  resolution:    76x76 dots per inch

i.e. on this machine, X uses 76 dpi for screen resolution.

This can be influenced by several means, but commonly it is taken from the
data the monitor provides, as it depends on its type and, for LCDs, the
panel used.
You should at least prepared to cope with different values for different
machines.
