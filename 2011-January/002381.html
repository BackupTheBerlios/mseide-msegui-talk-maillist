> I see it call dlsym procedure, what's library used?
For "reglang" the currently loaded language module.
If the given path has no directory loadlangunit() first searches the given 
library name in the locations defined by ldconfig. If not found it searches 
in the directory of the executable. Maybe the order should be reversed?
If the path has a library part it searches the given path only.
