> > Oo, I think it to replace standard font in app. As I know from your
>
> explanation, with this tool, we make options to our app to be customized by
> end user, and end users may change font standar with paramater
> --FONTALIAS..., right?
>
The IDE font alias options are used to define design time values for font 
alias which are registered at runtime. Assume we register a logical font at 
runtime with
"
 registerfontalias('acosysfont','Arial',16);
"
In order to use this font at designtime the IDE must know the accordant font 
properties -> 'Project'-'Options'-'Font Alias' must be defined 
for 'acosysfont'.

> My problem is, in Windows my app fonts is OK. But when I run it on Linux,
> the fonts is too wide.
>
Simplest solution is to run your application by a script with the 
appropriate --FONTALIAS parameter. Another possibility is to call
"
{$ifdef linux}
 registerfontalias('stf_default','',fam_nooverwrite,0,0,[],0.9);
{$endif}
"
in startup code before creating the forms. Replace "0.9" by your needs. 
Warning: the actually used font on Linux is distribution dependent.
