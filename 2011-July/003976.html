On 30 July 2011 12:09, Martin Schreiber wrote:
>
> Delphi 7 can handle multiple datamodules. I think it is not fair to present
> the brain dead idea to place all db components of a big application in a
> single datamodule as the "RAD way of development".

Then it must have been earlier Delphi versions that limited that
application to one datamodule. I only helped to maintaing it for 4
years. That project started long before I joined that company. The
"RAD way of development" encourages the developer to drop components
all over the place, write event handlers inside forms etc... This all
makes it very easy to create an unmanageable project very quickly.

I'm simply stating that taking a more well thought out design,
promotes the idea of an easier-to-maintain project - no matter how big
that project becomes. I have no doubt that a RAD style application can
be done correctly, it is just that IDE's like Delphi, Lazarus (and
probably MSEide too) makes it too easy for developer to go in the
opposite direction. Developing without thinking. That's only great if
you are doing prototyping.


> Think of a datamodule as a RAD equivalent of your business objects and there
> is no big difference anymore.

Honestly, I don't see that.
