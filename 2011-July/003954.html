If you used something like tiOPF (a Object Persistence Framework), that
would simply require a different compiler define and a recompile. NO
other source code or components need changing. tiOPF supports many
database backends using various DB components: CSV, Tab, XML, SqlDB,
ADO, DOA, BDE, UIB, Zeos, dBase, FBlib, IBO, FIB, IBX, IBO etc.
accessing database like Firebird, MS SQL Server, Oracle, Nexus, MySQL,
SQLite, FlashFiler etc.

  http://www.tiopf.com

tiOPF is hosted on SourceForge.net


> However all these databases requires the presence of an installed
> library (libmysql.so, libsqlite.so etc) to be installed on the computer
> running the program.

You can always implement a 3-tier (aka n-tier) system. That means your
client application would normally not require any database libraries on
the client side. tiOPF supports this too, via the remote persistence
layer which uses TCP (the HTTP protocol) and XML. Again, the tiOPF takes
care of the SQL->XML->dataset result transitions.

Other technologies that can be used for 3-tier designs are: WebServices,
CORBA, plain Sockets, custom HTTP+XML etc. Delphi used the proprietary
MIDAS for this.
