On 07/29/2011 02:03 PM, Marcos Douglas wrote:
> 
> tiOPF remote persistence supports all field types, even BLOB/IMAGE/TEXT?

Yes. TEXT and  Text Blobs  are handled as  standard text/strings. Binary
Blob, IMAGE etc  are handled via the  ParamAsStream() or FieldAsStream()
which MimeEncodes/Decodes the data  before/after transmission (just like
emails do with say image attachments).


> Which you work, tiOPF or tiOPF2? Or is this the same project?

I use  tiOPF2. There is a tiOPF3  branch already, but currently  that is
only for Delphi 2009 and up because we want to make that a fully Unicode
enabled  version. FPC doesn't  support Unicode  string types  inside the
compiler  yet,  so for  the  foreseeable  future,  I'll be  staying  and
maintaining tiOPF2.


What  I  didn't  touch  on  in  my  first  email  was  displaying  tiOPF
objects/data in a GUI. There are various methods that can be used.

 - tiOPF has  custom tiOPF-aware components. Currently only  VCL and LCL
   are supported.

 - Using  the tiDataset  unit will  allow you  to use  standard DB-aware
   components (even MSEgui DB components should work here I think).

 - Then  there  is also  my  preferred  choice using  Model-GUI-Mediator
   (MGM  is  a  variant   of  Model-View-Controller).  The  tiOPF's  MGM
   implementation  supports  the VCL,  LCL  and  fpGUI toolkits.  Adding
   support for MSEgui is planned, and should be rather quick and easy to
   do (if you  have MSEgui experience). The MGM mediators  are very well
   abstracted, so  most of  its functionality  is actually  non-gui code
   (thus working in any toolkit).
