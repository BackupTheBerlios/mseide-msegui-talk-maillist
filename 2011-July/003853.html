Hi Martin,

I have been using MSEide a lot lately. The session information is a real
nightmare for SCM (source repositories) systems. So this makes sharing a
project between developers, or working on the project from two different
machines a HUGE pain!

I don't have to actually change any settings in "Project Options".
Simply moving the IDE window around, reordering some files in the
editor. Close some files and open others..... All cause the *.prj file
to be changed. These are all "develop specific" preferences and
shouldn't apply to a shared file - used by many developers.

Can such information not be stored outside the *.prj files - some
"session information file" that I can tell the SCM to ignore. Because
currently every second commit in my repository is a *.prj file has
changed yet again. This is truly annoying and adds a lot of noise in my
project history (in the repository).

Clearly everybody here using MSEide are the only developers on each
project, never opening or closing files and never repositioning your IDE
windows. Or nobody here uses source code management (source
repositories)?? Or I simply don't know how to use MSEide "correctly". If
the latter, please give me some hints.
