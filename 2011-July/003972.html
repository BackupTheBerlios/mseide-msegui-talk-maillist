>  ZQuery1.Connection:=ZConnection1;
>  DataSource1.DataSet:=ZQuery1;
>  sqlsentence:=TstringList.Create;

In tiOPF that would translate to the following code:

uses
  …
  tiQueryZeosMySQL4,
  tiOPFManager,
  …

  gTIOPFManager.ConnectDatabase('server_ip:database', 'username', 'password');


The tiQueryXXX unit activates a specific storage backend, but this
could also be handled via other means like a command line parameter,
INI file, Windows registry setting etc. Same goes for the connection
criteria. Normally you wouldn't hardcode database location, username
and password information.


Now say for some reason you don't want to use the Zeos components any
more, but rather the SqlDB components included with FPC. Then simply
change the tiQueryZeosMySQL4 to tiQuerySqlDBMySql in the uses clause -
done! That's what I mean by "no vendor lock-in".
