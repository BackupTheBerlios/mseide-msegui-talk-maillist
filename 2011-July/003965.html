However my goal is more (also because I am not necessarily focusing on
RAD development only, but more "classic" applications, maybe even
without any gui, like web applications) to have only a few SQL
connections, and then simply prepare the SQL statement inside the program.

So essentially it is like this:

procedure connectbase;
begin
  ZConnection1:=TZConnection.Create(nil);
  Datasource1:=TDatasource.Create(nil);
  ZQuery1:=TZQuery.Create(nil);
  ZConnection1.Protocol:='mysql-5';
  ZConnection1.HostName:=DBHostName;
  ZConnection1.Port:=3306;
  ZConnection1.Database:=DBBaseName;
  ZConnection1.User:=dbuser;
  ZConnection1.Password:=dbpass;
  ZConnection1.Connected:=true;
  ZQuery1.Connection:=ZConnection1;
  DataSource1.DataSet:=ZQuery1;
  sqlsentence:=TstringList.Create;
end

procedure makesession;
begin
  sqlsentence.Clear;
  sqlsentence.Add('insert into session ');
  sqlsentence.Add('values ('''+cvalue+''', '''+currenttime+''',
'''+sessuser+''')');
  ZQuery1.ReadOnly:=false;
  ZQuery1.SQL:=sqlsentence;
  ZQuery1.ExecSQL;
end

etc, etc, very straight forward procedural programming.

You may call me old fashioned, but I like programming this way, and for
making web applications I consider it a great way not to lose the overview.

And when I am making GUI applications it can either be stand alone
versions (normally using DB-IV, which actually works quite good without
any additional libraries, but where I consider SQLite) or for a limited
number of clients, where I currently use MySQL, as I find it very fast
and robust.
