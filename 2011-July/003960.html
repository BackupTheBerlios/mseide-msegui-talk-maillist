On 29-07-2011 13:31, Graeme Geldenhuys wrote:

> tiOPF is hosted on SourceForge.net

As such i like the simplicity of using SQL sentences from Free Pascal,
and with Zeos it actually is very simple to handle.

As I see it, just from reading briefly, tiOFP is a totally other paradigm.

> You can always implement a 3-tier (aka n-tier) system. That means your
> client application would normally not require any database libraries on
> the client side. tiOPF supports this too, via the remote persistence
> layer which uses TCP (the HTTP protocol) and XML. Again, the tiOPF takes
> care of the SQL->XML->dataset result transitions.

Is that to be understood that one can make one binary, in which all the
tiOFP stuff is included, connecting directly to the database?

> Other technologies that can be used for 3-tier designs are: WebServices,
> CORBA, plain Sockets, custom HTTP+XML etc. Delphi used the proprietary
> MIDAS for this.

I looked at MIDAS a long time ago, but in my opinion solutions like that
is like having 6 gearboxes, 8 clutches, and a zillion of cam wheels
replacing the normal no nonsense transmission in a car.

It seems like the KISS principle is totally forgotten when it comes to
databases - imho the database should deliver what ever I ask it to
deliver, and the request for that should not be rocket science (and how
the database server then works internally is an other story, but that
should not be of my concern).

/Christian
