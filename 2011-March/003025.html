On Thursday, 3. March 2011 20.18:39 Julio Jiménez wrote:
> I also wonder why using:
>
> stockobjects.fonts[stf_default].name:= 'Helvetica';
>
> have no effect but if I change the font of a label and in name I type
> 'Helvetica' antialiasing is used. (but not, using stf_default even when I
> changed it to 'Helvetica') that's why I thought was using the font
> configuration wrongly.
>
stockobjects.fonts[stf_default] will be used if no widget or parent widget has 
an activated font property instance. If you want to redefine the mapping 
of 'stf_default' font name use registerfontalias().
