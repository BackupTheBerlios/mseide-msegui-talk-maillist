I think it's ok as it is actually... I suggested creartype as another
option.  MSEgui actually render fonts using antialiasing when windows is
able to do it. There was an error from my side thinking that MSEide was
using points for font size instead of pixels. Using the right proportion
they render in the same way.

If the system supports cleartype, the user can enable/disable it in desktop
preferences (screen preferences, appearance, effects...)

Could be interesting to have it as another option but not overwriting the
default one, I think. Windows Vista and up uses clear type by default so
MSEide uses it too (without extra options) if enabled in the OS.

The good thing is now we know how to enable/disable it in the software :)
