Op 2011-03-02 16:09, Christian Nobel het geskryf:
> So my question is, is it possible to embed libmysql.dll in the
> application, so I do not have to distribute it along with the program?

At least under Linux you can do the following:

  cat <file1> <file2> > combined_file


<file1> = you executable
<file2> = data file

If you know the size of your executable, you can find the offset where
the <file2> starts. You could then extract and save the data file as a
temporary external file and work from there. When you application
terminates, you delete the temporary data file. You could also read the
data file directly from the stream.

If this will work under Windows or with a shard library/dll, I have no
idea. But I do know reading data in this fashion (eg: an embedded zip
archive) is possible under Linux. Many of the Linux single file
installation setups (*.bin) work like this. A script in the front and a
data archive in the back.
