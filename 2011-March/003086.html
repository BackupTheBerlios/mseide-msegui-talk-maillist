> It seems Tahoma is not properly using antialiasing in MSEide+MSEgui.

Trunk 4377 has foo_antialiased2 in order to force using cleartype.
A MSEgui application can be forced to use CLEARTYPE_QUALITY independent of
the Windows system default setting by the commandline option
--fontalias=stf_default,,,,B -> CLEARTYPE_QUALITY
--fontalias=stf_default,,,,A -> ANTIALIASED_QUALITY
--fontalias=stf_default,,,,a -> NONANTIALIASED_QUALITY

Please test.
