The Windows remote access programs just grab what's been put on the screen,
usually compact it a bit, and run it through the wire. I.e., if there were
a couple changes (or more) in the meantime, you won't have an indication
that something went on. You see a series of still photos, so to speak.

X11 _can_ transfer all the dynamics over the line, and still allow the
client to not being hampered by line speed, if drawing is done
asyncronously. You get to view every picture of the film, metaphorically.

If the client instead waits for every primitive to be confirmed completed,
anyone will be able to imagine how that performs. And - as "they" say -
_that_ is the reason why the "modern" big desktops show such bad
performance. The implementors avoided a little basic work to ensure fast
operation, and leave the user with painfully slow painting activitiy.
(Although the painting itself usually is "fast enough", but the
interactivity, the reaction speed, suffers from the huge overhead.)
