On 18 September 2010 02:09, Ivanko B wrote:
> even documenting which changes made to a project  (a kind of detailed
> Changelog) is very boring & time consuming if there're a lot of

I found a quick and easy solution to that... simply add more detailed
commit comments in your code repository / SCM. What is strange, is
that on average most SVN users tend to add a one line cryptic commit
comment which means nothing 2+ months later... Git users tend to add
more descriptive commit comments (one line summary, blank line, then
more detailed commit comment). I use the latter, and it works really
well if you need to generate a Changelog or review old changes. It
hardly takes up any extra time too.
