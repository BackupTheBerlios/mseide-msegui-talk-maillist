Well, I'm curious enough to cross check ASAP, so I made a couple tests and
found that this strange behaviour
- for one alos occurs with an even more simplifiedform containing just one
  (close) button, but
- for another, occurs onlywhen the opnebox window manager is also active.
I.e., the problem is caused by openbox, of which version 3.3.1 is installed
on the target machines, while my "big" machine runs version 3.4.11.2.
But even a much earlier version (after 3.3.1) would require me to update
the whole system, as the dependencies reach back to libc...
So the alternatives seem to be to update the whole thing (takes quite a bit
of time) or run without window manager, which should be feasible for the
target application - I just have zo make sure it will suffice.
