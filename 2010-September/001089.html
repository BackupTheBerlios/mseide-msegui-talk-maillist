I must agree with Martin. Documentation should not exist in the source
code. Minor technical comments are a different story, but that is not
documentation.

To make documentation easy, no matter the format, there needs to be
some integration with the editor or IDE. Lazarus IDE does this - you
can document classes while you code, and as the cursor moves through
the code, so the documentation window is updated. This is very handy.

For fpGUI, I simply edit the fpdoc description XML files inside the
IDE. I have some code templates setup to create the most used XML
notes (unfortunately not available when I use MSEide). fpdoc
description XML files actually have a very simple syntax - that means
a lot coming from me, because I hate working with XML. Anyway, simply
editing the XML files every now and again, while coding is not really
that bad - slowly but surely the docs for fpGUI are growing. :-)

Anyway, to give a good example of documentation inside the source code
not working - obfuscating the source code, is with the tiOPF project.
We initially used pasdoc, and later the code was so hard to read,
because there was so much docs. Since then we started moving those
docs into fpdoc XML files and things are slowly getting better. Plus,
fpdoc supports many output formats, can include images and source code
examples etc... a much better deal in the end.
