Hi Martin,

Attached is a quick console application I put together to generate
fpdoc style documentation from MSEgui sources. This might be useful
for someone to build further on. I have included in the builddocs.pas
unit header some TODO list items I think can be added.

What does it do?
'builddocs' has a couple of command line parameters that could be
extended. Currently it simply generates a unix script, which in turn
can be run to generate a msegui.ipf (INF help source file). It has two
modes. Build build a script only for units that are documented (that
have .xml description files), or build a complete skeleton help of
MSEgui.

In also reads the a 'fpdoc' environment variable, so is flexible to
switch between fpdoc versions. Currently INF docs can only be
generated with the fpdoc found in FPC 2.5.1 (trunk). The FPC 2.4.x and
earlier fpdoc versions are severely broken.

What could be added?
* Generate Linux shell scripts and Windows batch files
* Use TProcess to run fpdoc directly, without the need for a script
* more parameters to toggle between the various fpdoc output formats.

I think this file console app source code should live in the
<msegui>/help directory, but ultimately, it's up to Martin.

Sample XML description files:
I have also included two example XML description files, which I used
to generate the sample INF help file (posted earlier). Again, I think
it is important that such help files should stay close to the MSEgui
source code, thus I think it should be located in the
<msegui>/help/xml/ directory. This would mean anybody that has MSEgui
source code, can easily find the existing help, and maybe make
contributions. Also they can easily build there own help output, using
builddocs + fpdoc.

Anyway, the files are attached in the tar.gz archive - do with them as
you please. ;-)
