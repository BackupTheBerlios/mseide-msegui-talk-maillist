Hi Martin,

Sorry for all the mails, but I expect you would like feedback on Code
Templates. :-)

Below is a code template for inserting a fpdoc seealso documentation
tag when writing xml description files.

-------------------[ fpdoc_sa.mct ]-------------------------------
[header]
name=docsa
comment=Documentation SeeAlso tag
noselect=1
cursorcol=22
cursorrow=1
params=1
 linkid
[]
<seealso>
<link id="${linkid}"><link/>
</seealso>
-----------------------------------------------------

A few issues I found:

1) cursorrow seems zero based, but the MSEide and all other editors,
the first line in a text file as show to the user is line 1, not line
0.

2) placing the cursor is quite problematic, especially if parameters
are used. Depending on the length of the parameter value, I can't
foresee the cursorcol position. So whatever I set it at, it will
always be wrong because the parameter value can be different every
time.
I would maybe suggest using the same cursor position method used by
Delphi and Lazarus. The | symbol added in the code template text
denotes where the cursor will be placed (replacing the | symbol). This
should elimenate cursor placement issues with parameters. For example,
fpdoc_sa.mct could be reduced to the following:

-------------------[ fpdoc_sa.mct ]-------------------------------
[header]
name=docsa
comment=Documentation SeeAlso tag
noselect=1
params=1
 linkid
[]
<seealso>
<link id="${linkid}">|<link/>
</seealso>
-----------------------------------------------------

No need for cursorcol and cursorrow anymore. This seems more intuitive
and less error prone. This also reduces the effort of having to count
line numbers in large templates so I know what line number to use.

3) noselect=1 should probably be the default, instead of noselect=0.
This would reduce the effort of having to insert that line in every
template file.

4) While MSEide is open, I edit one of my *.mct files. The newly
modified code template does not take effect until I restart MSEide.
MSEide seems to cache the template content or something. Maybe it
should only load a template once Ctrl+T is pressed, and not at IDE
startup.

5) A nice to have.... Could you supply a syntax highlighter for *.mct
files by default. Only the [header] block must to by highlighted. This
would round off Code Templates nicely. ;-)
