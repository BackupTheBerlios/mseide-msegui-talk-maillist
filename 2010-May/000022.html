On Tuesday 18 May 2010 14:42:25 Graeme Geldenhuys wrote:
> Hi Martin,
>
> Any reason why MSEgui doesn't use the TBasicAction class (included
> with FPC) as a starting point for MSEgui actions? As you described in
> the other message thread about activators - that is functionality the
> TBasicAction class already supports. Maybe MSEgui code pre-dates FPC's
> TBasicAction (fpc's subversion repository only goes back to 2005 so I
> don't know the actual date TBasicAction was implemented)?

I don't remember if it already existed and worked when I implemented MSEgui's 
action system. Basing own functionality on FPC technology is problematic 
because it is normally not possible to change FPC code in reasonable time and 
with reasonable energy except if FPC is not Delphi compatible. ;-)
MSEgui taction concept is very different from Delphi.
A good example are datasets where MSEgui still inherits from TDataset and 
TField for easy integration of foreign components. Because of that MSEgui 
needs cracker classes in order to access private class elements for 
workarounds and extensions and the design flexibility is limited.

Martin
