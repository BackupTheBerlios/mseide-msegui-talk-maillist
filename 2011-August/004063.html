Hi Martin,

After the discussion in Lazarus mailing list, I think there is one
thing that would make the Ctrl+E (opened file grid) much more usable,
even for 400+ units.

Currently you can open than window and start typing. Focus will jump
to the unit that starts with what you typed.

Why not allow the matching to occur anywhere in the filename, not just
from the start. Also maybe filter the displayed items as you type. So
if you have 400 units open and you type 'learner', it will filter the
list down to any units that contain the string 'learner' somewhere in
the filename. Backspace will undo the filter one character at a time.

You said 400 units are too much for MSEide, but I have 2 projects that
I work on daily that have more units than that. The Project Tree
doesn't help much in those cases, because there are just too much
scrolling (not enough vertical screen space). So the Ctrl+E seems the
only logical way to quickly find a unit.
