On 08/02/2011 03:00 PM, Marcos Douglas wrote:
> 
> You already talked about your report engine before... but is private, right?

No, the reporting engine I use in also in tiOPF. Search for tiRTFParser
(original VCL version), or tiRTFReport (fpGUI version). It uses RTF
(Rich Text File) files as a report template. You can use OpenOffice for
example as your report designer. I even use OpenOffice (in read-only
mode) as our report viewer. OOo is nice in read-only mode because it
hides all edit toolbars and menus and looks like a "viewer".

tiRTFReport supports TDataset, tiOPF Objects or custom input. Images are
support too. It has many built-in functions, and you can extend/add new
functions very easily using Object Pascal.


> At work, with Delphi, I use ReportBuilder but just with TDataSet.

tiOPF has a unit for ReportBuilder support as well - reporting from
TDataset or tiOPF's objects directly.


> If the MGM implements an OnChange event, I can not implement my own
> OnChange event in the same component?

MGM would use the OnChange event yes, so if you wanted to do more
processing, then simply override the one in the MGM mediator.


> Sorry, I did mean "1 thousand of lines".

I have no idea, but I would imagine they will be similar in memory usage.


> I liked your approach using Hint property, thanks!

Screen space is limited, so Hint's looked like the most logical thing to
use because it doesn't take up extra space. We also don't like to
interrupt the end-user with popup windows - that is just bad practice.


> You need to codify all TtiObject's descendants and their attributes,
> right? I don't like it.

I auto-generate the initial units (classes and CRUD visitors), then
simply maintain the by hand from then onwards. There are two tools that
could be used for this tiSQLEditor (available in tiOPF) and
LazDataDesktop (included with Lazarus - but is a stand alone product).


> In my classes I try use the DRY principle, like Ruby ActiveRecord, and
> get all columns (attributes) dynamically.

There is some sample code in the Quarantine directory what "Value Types"
are used. I believe there is also an example on property attributes eg:
 MyObject.Attribute('Address').Value
I personally don't use either of these.
