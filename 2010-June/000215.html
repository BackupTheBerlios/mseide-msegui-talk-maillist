Sorry for responding twice to this message - I think I have some news. ;-)

_This_ posting should reach the list via NNTP now, at last. (I hope it
will.)
It seems that the reason for _my_ inability to post was a misunderstanding
of the "authentication request" email sent by gmane.
You have to respond to the authentication server itself, whose address
should be stated in the email's "From:" header. The content of the response
is irrelevant.

I was deceived by the wording of the request, which does suggest to me that
one had to send a message _to the list mangement service_ address as given
there. It does nowhere say one should respond to the "authorization"
server itself (which doesn't provide a full authorization check anyway, just
a weak verification of the prospective poster).
Maybe this can save someone else some headache and annoyance...

(BTW, Lars stated the gmane service would _not_ implement the "IHAVE"
command of NNTP - should a client insist on using this command for posting,
it _will_ fail, then. My snnews seems to be able to get along without.)
