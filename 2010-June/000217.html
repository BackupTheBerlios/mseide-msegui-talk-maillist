Zsolt Szokolai wrote:

> Hi all,
> how should I use the tmsesqltransaction around tsqlresult?
> In strictly read only case.
> 
A transaction is always necessary.
In order to use implicit transactions activate tmsesqltransaction.options
tao_fake. tao_fake does not work with Firebird AFAIK.

> For example:
>  transaction1.active:= true; <-- is this needed at all?

Not necessary because tsqlresult sets it automatically if activated.

>   tsqlresult1.active:= true;
>    foo:= tsqlresult1.cols[0].asinteger;
>    bar:= tsqlresult1.cols[1].asinteger-1;
>   tsqlresult1.active:= false;
>  transaction1.active:= false; <-- and this?
>
Necessary if tao_fake not set. Not necessary if you want to fetch more data
within the same transaction.
