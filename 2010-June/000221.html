> I set up a dbgrid

(tmsepqconnection-tmsesqltransaction-tmsesqlquery-tmsedatasource-tdbwidgetgrid-sometdb*edit)
> I took care of selecting the pkey in sql and dso_autoapply and
> dso_autocommitret. It's works fine.
> But what is transactionwrite for?
> Is this use another transaction for write?
> 
Yes. The insert, update, and delete statements use
tmsesqlquery.transactionwrite if set, the fetch statement uses
tmsesqlquery.transaction. It sometimes needs less server resources to use a
longtime readonly transaction for data fetch and short time
writetransaction with dso_autocommit fot data update.
