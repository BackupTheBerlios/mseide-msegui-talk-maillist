Hi Martin,

For a while I have switched back my keyboard layout to "US Dvorak", but I
found it is not optimized for programming, and I can't type the numbers
very fast. It does make integrating keyboard shortcuts with applications a
bit easier though.

Anyway, I switched back to my preferred "Programmer Dvorak" layout (now
comes standard with Ubuntu for a few releases already). Setting up bookmark
keyboard shortcuts in MSEide has a few issues. See attached image for what
I tried. The numbers in the screenshot is not self explanatory, so I'll
explain how I setup the shortcuts.

Setting bookmarks I used:
  *  Ctrl+Shift+<number from numpad>

My numlock key is on, so if I'm in any editor and I press the numpad key 7,
I get the number 7.

Clearing bookmarks I used:
  *   Ctrl+Shift+<numpad minus key>

To jump to bookmarks I used:
  *   Ctrl+<what is a normal qwerty keyboard's top number row>

So what is Ctrl+1 or a qwerty keyboard will be Ctrl+& on the Programmer
Dvorak keyboard. My buttons cap on my keyboard are not changed, so to me,
it looks like I am pressing Ctrl+1 (my keyboard is really a qwerty
keyboard, just layout is changed via software), when in fact it is Ctrl+&.
In Programmer Dvorak, the numbers in the top row is accessible via Shift.
Just pressing the top row actually gives the symbols: eg: &[{}(=*)+


The problem #1:
  * Setting up the shortcuts in MSEide seem to work fine, and no
    conflicts are shown. Again, see attached screenshot.
  * No while I am in the MSEide editor, I want to set the number 7
    bookmark, so I press Ctrl+Shift+<numpad 7>
    In the editor, all text is now highlighted, and the editor jumps
    to the first line of the unit.

So it seems like there is a "source editor widget" key conflict, that is
not shown in the MSEide shortcuts dialog.

Clearing a bookmark with my shortcut setup seems to work. Also jumping to a
bookmark seems to work with my setup.

The problem #1:
 * Seeing that jumping to bookmarks work, I thought I would use the top
   number row for setting bookmarks too. In Programmer Dvorak they aren't
   really numbers, but remember my keycaps are not changed, so looking at
   they keyboard, it looks like numbers.
 * Again, setting up the shortcuts it the IDE seems to go well.
 * Setting bookmarks 1, 2, 3 seems to work well.
 * Setting bookmark 4, actually moves and sets bookmark 3.
 * I thought I made a mistaking in the Shortcuts dialog, but in fact I did
   not. No conflicts are shown, and the hex values are different for each
   shortcut, get bookmark shortcut for 3 and 4 seem to be the same for
   the source editor.


I'm currently using Ubuntu 10.04 (with all latest updates installed).
Programmer Dvorak is standard with Ubuntu since 8.10 I think.
