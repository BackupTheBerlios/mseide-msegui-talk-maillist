BTW, I cannot in any respect give you information, not even own opinion,
about the system prepared / required directory hierarchies under any of the
current Windows. You probabely should check in the corresponding groups,
even "de.comp.lang.delphi.misc" carried some very interesting information
already (but I didn't save it for lack of relevance).

Judging from your mentioning of "tstatfile.filedir" I would suppose this
might be a directory for storing more or less user dependant data - in this
case, under any unixoid, the user's home directory should be the first
target, and the file(s) or directory created should be made "hidden" by
the common convention of prefixing the name with a period (".").
You might consider to create an application specific subdirectory if there
were several files to save, or use one of the desktop environment prepared
subdirectory trees - though the latter should be restricted to programs in
fact depending on the respective environment, because it might happen that
someone changes to a different one, thereby completely removing the
previous environment's hierarchy.
BTW, the "~/.config" hierarchy seems to belong to the "open desktop"
specification newly introduced lately.
