Well, and the answer, as stated, is:
No, it shoudln't usually, except it is completely certain that the
application using the mechanism may _never_ use more than one single
configuration file. In any other case the path should _also_ contain the
binary's name as its last element, to be used (and created) as a
subdirectory, "<home>/.config/<application name>".
And as another condition - although I cannot state that with any authority
- the application ought to conform to the open desktop standard definition.
If you want to drop the last requirement, you should simply not use the
".config/" part of the path, all other arguments staying the same.
On Linux, you _have_ choices.
