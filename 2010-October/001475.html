  On 15.10.2010 07:55, Martin Schreiber wrote:
> Additionally the goals of MSEgui and FPC are different so we won't get
> optimal solutions for MSEgui.
While I do see different goals/directions between MSE and Lazarus, I 
don't see this between MSE and FPC:

AFAIK, MSE is supposed to always use the latest release of FPC (or do 
you intend do fork FPC itself or the FPC RTL for better use with MSE).

Regarding the subject: the MSE Event queue solution is superior to the 
one done in Lazarus, while FPC does not provide any at all. So moving it 
to the FPC RTL seems to be "globally" the best way to go,

I suppose that the FPC team would be very happy if you would agree to be 
the maintainer of the Event Queue implementation in the RTL. Of course 
there would need to be a dedicated documentation for this. (I would be 
happy to crate it.)

I don't think that it would be a problem if you would enhance the 
implementation for MSE's benefit as long as it still adheres to the 
original documentation.

I suppose additionally there would need to be testing programs (maybe 
same need to be able to be started automatically with an evaluation 
process). I would be happy to help creating those).

-Michael
