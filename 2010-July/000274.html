> What is the "persistent field" at all?
> And where and how can I set up it?

By default tmsesqlquery creates fields for all fielddefs the query returns
after open. On close the fields are destroyed. Because of that the
properties of the fields can not be stored in *.mfm file which means they
are not "persistent".
In order to have fields which are editable by the objectinspector and store
their properties in the *.mfm file one can use the field components from
tab 'DBf'. It has the advantage that the fields can be accessed by the
compiler which has less overhead than "fieldbyname".
Another possibility is to use internal persistent fields
(tmsesqlquery.controller.fields).
The objectinspector provides a field editor, click on the ellipse of
tmsesqlquery.controller.fields.count.
