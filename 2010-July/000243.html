Zsolt Szokolai wrote:

> How can I use the SQLInsert in tmsesqlquery?
> I have a table with an autoincrement integer primary key:
> 
> CREATE TABLE utemmegrend
> (
>    id integer PRIMARY KEY DEFAULT nextval('sequence'),
>    blah boolean,
>    ...
> )
> 
> The problem is the error message at new record post:
> An error occured while applying the UPDATES in a record:
> tmsepqconnection1: Execution of query failed
> (PostgreSQL: ERROR: null value in column "id" violates not-null
> constraint)
> 
> I suspect it's resolvable by own insert SQL but how?
> How can I retrieve the values from the actual row which are entered by
> user for build my own insert statement?
> Via parameters?
> params[0].asinteger:= editwidget.value?
> 
Use a tsequencelink (tab DBf) in order to automatically fetch the value from
the sequence generator and store into the "id" field before inserting a
record.
Or if you don't need to know the value of the "id" field exclude it from the
select statement or make "id" a persistent field and deactivate
TField.ProviderFlags pfInUpdate.

Martin
