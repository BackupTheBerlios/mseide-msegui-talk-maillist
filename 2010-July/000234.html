Zsolt Szokolai wrote:

> How can I set up a coded field in tdbwidgetgrid?
> For example an order table which has an integer filed for customer id.
> Another table contains the customers with name address etc.
> I'd like to see a dropdown cell where I can select a customer.
> Then the selected customers id enter into order.custid field.
> I suspect that t{|db}dropdownlistedit{|db|lb} controls for this, but I
> don't feel the differences and usage.
> What is the simplest way achieve this input? (and output, because a
> want to see customers names, don't ids, of course)
> 
Use a tdbenumeditdb for an integer key, tdbkeystringeditdb for a string key.
datalink.datasource and datafield select the lookup field in the master
dataset.
dropdown.keyfield selects the keyfield in the detail dataset.
dropdown.datasource and cols[n].datafield define the fields listed in the
dropdownlist. The field of dropdown.cols[dropdown.valuecol] will be
displayed as edit text and in the dbwidgetgrid column.

Better performance for very big lookup recordcounts can be achieved with
tdbenumeditlb and a tdblookupbuffer.
