> Why do use use tsequencelink (wanna know exact sequence values, me
> guess ) ? It's mainly needed for 100% safe handling master-detuil
> table relationships since the DB facilties (nextval/currval)  is
> unreliable here (other DB clients may affect nextval thus currval). If
> Your case isn't master-datial then You may use the DB facilities:
>
> CREATE SEQUENCE uz_provinces_id_seq;
>
> CREATE TABLE uz_provinces (
>    id INTEGER PRIMARY KEY DEFAULT nextva('"uz_provinces_id_seq"'),
>   code INTEGER,
>   name TEXT,
> ...
> );
>
> INSERT INTO uz_provinces (code, name) VALUES (1, 'Tashkent'); /* no ID
> vale supplied - it'll be read from the bound sequence */
>
> It will generate a new value sequence value when each inserting -
> exactly what TYou need :)
