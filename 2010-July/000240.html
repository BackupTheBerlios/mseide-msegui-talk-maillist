Yes, maybe rightly so, if you were equally deceived by the authorization
email as I was - the mail that begins so:

"You have sent a message to be posted on the
 gmane.comp.ide.mseide.user newsgroup."

and ends with:

"You have to respond within one week.

 -- 
 Your friendly autoauthorizer at Gmane.org
 http://gmane.org/"

You have to respond to _this very message_, using the "From" address given
there (something looking like
 "Gmane Autoauthorizer <auth-[garbage]@auth.gmane.org>",

_not_ any of the list addresses!

If you did this already, and received a response like

"This message was created automatically by mail delivery software (TMDA).

 Your confirmation was accepted, and so your original message has been
 delivered."

then the following may apply to your NNTP client software:

> > (BTW, Lars stated the gmane service would _not_ implement the "IHAVE"
> > command of NNTP - should a client insist on using this command for
> > posting, it _will_ fail, then. My snnews seems to be able to get along
> > without.)

BTW, if you missed the authorization deadline, you should be able to invoke
another authorization request by just trying to post another message to the
_news group_. This gets you another week to respond.
