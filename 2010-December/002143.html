Found :

//--------------------
rc:= SHGetFolderPathW(0,shortint(CSIDL_FLAG_CREATE or
CSIDL_APPDATA),0,0, at buffer);
//--------------------

works well !

CSIDL_FLAG_CREATE ($8000) is much-much large value than CSIDL_APPDATA
($1A) & possibly overwhelms that on ORing

For fpc 2.4.3, explicit typecast  is needed ("C" uses "int" here).
Possibly,there're some changes in passing STDCall args to win-32 API. 
And it seems that plain usage of non-type constants for win-32 API
args  is a bad idea.
