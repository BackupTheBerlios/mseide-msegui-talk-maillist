Hallo Sri,

Du schriebst am Fri, 24 Dec 2010 00:44:02 +0700:

> My purpose is to run my app without any other app, just only my app.
> I'll use it to POS (Point Of Sale).
> Note, it just my experiment.

This is an experiment for you - X11 is built so this will work. _Any_ X11
application can be used so. Even a plain Xterm.

> My app doesn't have window title, window buttons

Those "decorations" are some of the things window managers are meant to
cater for. Other such things are moving and sizing windows, this can get a
bit awkward without a window manager.

> (close,minimize,maximize). It's amazing for me because App built with
> MSEgui doesn't required any GUI library.

It _does_ use one: Xlib and its subordinate libraries.

> What app should I install to create window manager?

Ermmm - a window manager, perhaps? There are so many to choose from...
BTW, I'd like to throw in openbox and wmx, both very basic ones. Contrary
to Graeme's favourite, they don't even have a concept of a "task bar" -
what the heck should _that_ be?  ;-)

> BTW, than I think, it's very possible to create a Linux Distro with
> MSEgui, maybe it can be expereimental only and for hobbiest only.
> So, there will be a Linux Distro with FreePascal language. :)

If you want to write the pertinent ancilliary programs for it, certainly.
Or, rather, _any_ distribution can use any MSEGUI application, provided the
X11 system is installed and works.

> I think only MSEgui and fpGUI can run inside this distro.

Nope. Any program, even text mode programs, will run that way - the latter
need a terminal program as a host for in- and output, but they will run.
And MSEGUI programs, like e.g. MSEIDE, will also run, of course.
