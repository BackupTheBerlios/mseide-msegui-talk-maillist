I don't found implementation of isolation level in tmsesqltransaction.
Currently in my app, if others user/connection change some data, I
can't read the changes until I refresh transaction. But refresh
transaction make other effect, such as refresh dataset that I don't
want.

I read that transaction of db engine have some isolation level method:
- Read Uncommitted
- Read Committed
- Repeatable Read
- Serializable

The documentation about this :
MySQL at http://dev.mysql.com/doc/refman/5.5/en/set-transaction.html
FireBird at http://www.firebirdsql.org/dotnetfirebird/transaction-isolation-levels.html
PosgreSQL at http://developer.postgresql.org/pgdocs/postgres/transaction-iso.html
