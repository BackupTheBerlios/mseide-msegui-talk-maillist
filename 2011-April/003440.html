> Correct, the searched column is always the first column. Can be used for
> example to select items by numbers, valuecol = 1:
> |COL0              |COL1
> ----------------------------------
> |1 the first row   |the first row
> |2 the second row  |the second row
> |3 the third row   |the third row

Is also ok when one knows it, but confuses a little bit, because one
would think that the valuecol was determining.

Eg. I have a dropdown where I have a customer index and a name:

Index     Name
------------------
1         Smith
2         Anderson
3         Wayne
4         Dallas
5         McClane

Here it has to be done like this for the incremental search to work:

Name      Index
------------------
Smith     1
Anderson  2
Wayne     3
Dallas    4
McClane   5

Instead of just saying that the valuecol also per default was used for
the incremental search - could also be expanded if the dropdown included
both first name and surname.

Or maybe even smarter, defining which column to use for search.
