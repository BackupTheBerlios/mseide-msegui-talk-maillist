> Please don't use "i386-win32" & "i386-linux" subdirectoris within "addon"
> directories - it significally complicates auto bulding.
> You may use {$ifdef} in same level files instead.
>
The tosprinter (OS specific unit) is inherited from tprinter (general
unit). And it's difficult to switching between printer library in
linux (cups) and windows (gdi).

Please you add ${MSEDIR}lib\addon\osprinter\$TARGET\ to project make directory.
I use this script to build from script:
-Fu${MSELIBDIR}addon/osprinter/i386-win32/
-Fi${MSELIBDIR}addon/osprinter/i386-win32/
