On Thursday, 17. February 2011 10.30:34 Graeme Geldenhuys wrote:
> Op 2011-02-17 10:58, IvankoB het geskryf:
> > Does the above mean that all MSEgui (FPC,Lazarus etc ) based apps MUST
> > be GPL-ed
>
> NO. Some parts in FPC and Lazarus (eg: the RTL) is LGPL with a static
> linking exception. Meaning you can use that code inside your application
> (not a external library), and still do not require to make your
> applications GPL.
>
> I believe MSEgui has a similar license to the above.
>
"
As a special exception, the copyright holders of this library give you
permission to link this library with independent modules to produce an
executable, regardless of the license terms of these independent modules,
and to copy and distribute the resulting executable under terms of your 
choice,
provided that you also meet, for each linked independent module, the terms
and conditions of the license of that module. An independent module is a 
module
which is not derived from or based on this library. If you modify this
library, you may extend this exception to your version of the library, but you 
are
not obligated to do so. If you do not wish to do so, delete this exception
statement from your version.
"
The exception especially means that you don't need to provide a possibility 
that the user can use  a newer version of the library, I.E you don't need to 
distribute linkable object modules of your software as LGPL requires:
"
    b) Use a suitable shared library mechanism for linking with the
    Library.  A suitable mechanism is one that (1) uses at run time a
    copy of the library already present on the user's computer system,
    rather than copying library functions into the executable, and (2)
    will operate properly with a modified version of the library, if
    the user installs one, as long as the modified version is
    interface-compatible with the version that the work was made with.
"
Copy pasting library code in a distributed non open source project is not 
allowed.

> Lazarus IDE (and I believe MSEide) are GPL applications  (different
> license to the gui library parts) - they must always be free and open,
> even for derived work based on their code.
>
Correct, MSEide is GPL.
