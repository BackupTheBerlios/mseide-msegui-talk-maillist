As I can see, this primarily applies to how the image is shown in the
application, not the size of the image as such.

My idea was to change the size of an image, eventually loaded as a
stream from a blop field in my MySQL database, resize it, and store the
new picture in another blop something like this:

procedure tmainfo.resizepic(const sender: TObject);
var
Jpg: timage;
Stream: TMemoryStream;
begin
Stream := TMemoryStream.Create;
Jpg := timage.Create;
TBlobField(tmsedatasource1.dataset.FieldByName('img')).SaveToStream(Stream);
Jpg.bitmap.LoadFromStream(Stream);

***** Here I would like to do something like: Jpg.resize(10%); *****

tmsezquery1.edit;
TBlobField(tmsedatasource1.dataset.FieldByName('thumb')).LoadFromStream(Stream);
tmsezquery1.post;
jpg.Free;
Stream.Free;
end;

I could of course save the file on disk, and use an command line tool 
like ImageMagick, but it seem to be a cumbersome way to go.

Another problem I have faced in my testing is (if we look apart from the 
nonexistent image.resize) that the program compiles ok, but if trying to 
run resizepic I get an error:

Graphic format error (probably the Jpg.bitmap.LoadFromStream(Stream) is 
the culprit, as the remaining streamhandling works.
