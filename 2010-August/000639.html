An IDE is at least also something like a make program - it controls the
build process, calling any needed programs to effect this, and gathers the
output (and return codes) for evaluation by the user, _and_ attempts to
provide information to the editor to allow for navigation by "compile
problem".
BTW, for Dos, I once had an editor (a highly programmable one) taht could
act as a full fledged IDE also. ;-)

> > (albeit regrettably few) that allow you to use your favourite editor.
...
> Quite simply because not all editors can be embedded in another
> application or controlled from a remote application/process.

The minimal interface for such use would be to run the editor with a source
file as a parameter. Any auxiliary information could in principle be
presented in a non-editable vieweer application, Any editor schould be able
to support this; of course, a programmable Editor will help a lot in this
setting.
