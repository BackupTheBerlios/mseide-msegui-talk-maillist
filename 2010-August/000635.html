The behaviour I described towards the end is even quite simply implemented.

For jumping ahead:
 1.- See if you're at EOF. If so, exit.
 2.- Skip to the next character
 3.- See if you're at EOF. If so, exit.
 4.- Check the current character.
     If it is a letter character, and the character immediately preceding
     it is a separator character, you're done - exit.
 5.- Repeat at 2.

For jumping backwards:
 1.- See if you're at BOF. If so, exit.
 2.- Skip to the previous character
 3.- See if you're at BOF. If so, exit.
 4.- Check the current character.
     If it is a letter character, and the character immediately preceding
     it is a separator character, you're done - exit.
 5.- Repeat at 2.

Tremedous, isn't it? ;->

(BTW, separator characters are of course whitespace, line endings and
punctuation; maybe also special characters like '%' or '§' and non-printing
characters, if used. Digits and probabely '_' should be considered word
characters, I think. [Yes, certainly, alphabetics too, of course ;) ])
