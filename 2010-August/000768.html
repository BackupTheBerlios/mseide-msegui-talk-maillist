> Warning: tTimedDialogfo.Over is nil!

Right, this is "as designed", as you say in such cases.
This is because it has dual uses: For one, the user can assign an already
existing timer to it to be used by the component, or if this isn't done,
the component will instantiate one itself as soon as it will be used:

  FUNCTION tTimedDialogfo.Execute (...
...
     IF NOT Assigned (Over) THEN Over:= ttimer.Create (Self);

This will (should) be destroyed automatically when the component is taken
down, as it is its owner then, in contrast to the case of a user supplied
timer (which will persist). It seemed to work as intended - doesn't it?
