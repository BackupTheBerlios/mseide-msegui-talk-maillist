3-4 years ago I created a component called Mozilla Plugin Panel. It's a
panel component that can host any Mozilla/Netscape Plugin. I got PDF and
Flash to work under Windows, and Flash partially working under Linux (my
knowledge of Linux X11 was very limited back then). What the component
does, is fake the Mozilla Web Browser interface that a plugin requires, I
then give the plugin a canvas to paint on. In theory any Mozilla Plugin
should be able to work, even Java. I got the idea from the way we used to
embed Flash OCX in our Delphi apps. We needed a cross platform way of doing
something similar.

Unfortunately I got pulled off the project before I had time to complete it
(we went for a CGI app instead of a Desktop app in the end). But I hope to
one day finish it up.

More details can be found in the following URL, and a link to the source
code repository. Patches are of course welcome. ;-)

  http://opensoft.homeip.net/wiki/wiki.cgi?p=mozilla-plugin-panel
