Graeme Geldenhuys wrote:

> On 18 August 2010 11:45, Martin Schreiber  wrote:
>>
>> It sometimes happens for unknown reasons.
>> "not decorated" exception means that the window manager does not reparent
>> a new window into a container window with window manager frame elements
>> in a given time. A WM theme with many long running motion effects at
>> window creation can produce the error.
> 
> OK, that doesn't sound too serious then. :)  It was probably caused by
> my many failed attempts to dock the Messages window under the Editor
> window.  I got that docked layout write with a previous project (Main
> window at top, Project Tree on left, Editor on right and Message at
> bottom), but for the life of me, I couldn't duplicate that docked
> layout with this project.  I'm still learning MSEide. :)
> 
> 
- 'View'-'Panels'-'New Panel'.
- Drag the editor window by the grip into the new panel.
- Drag the message window by the grip into the new panel.

If the editor window is already docked drag the message window to the grip
of the parent panel of the editor window. Dragging to areas other than the
grip docks onto the top level panel.
