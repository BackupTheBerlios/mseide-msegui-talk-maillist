Julio Jiménez wrote:

> It seems I'm wrong in some concepts. I guess find does binary search using
> local indexes
Correct.

> but how does work locate? sequential search (not using 
> database engine)?
> 
locate() uses sequential search from current record with TDataset.Next and
Prior.

> About isnull vs nil for null search value, using nil in 'avalues' looks
> better from my point of view.
> 

I think now the "isnull" parameter is necessary because without it needs a
different locate() call for every key/NIL combination, the type of the
keyvalue (actual value or NIL) must be known at compile time. The "isnull"
parameter can be an empty array, in this case it will be ignored. I'll add
the NIL-pointer method to find() also so we have both possibilities in
find() and locate().
