Are conditional breakpoints implemented in MSEide? See attached
screenshot. I set a breakpoint with a condition. I don't know the
format of a condition expression, so typed what I thought might be
correct: TypeDecl.Name = 'TRGBTriple'

I added my run parameters into the Target > Environment dialog. Run
the application, and the program stops at the breakpoint. I added the
extra s variable into the code so I can watch it easier. It seems that
the breakpoint condition was not actually used, it simply stopped

FName field variable directly, so that shouldn't be the usual debug

to type the Condition, so that the program only stops when the
