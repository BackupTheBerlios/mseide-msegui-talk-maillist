Hallo Graeme,

Du schriebst am Sat, 14 Aug 2010 15:41:48 +0200:

> * word jumping: When you navigate words by pressing Ctrl+LeftArrow,
> MSEide's editor places the cursor on the last character of the
> preceding word. All other editors place the cursor before the first

"All other editors" is a _very_ strong assertion, so strong that is usually
false. As it is here.
The very editor I write this response with does it differently: it puts the
cursor _on_ the _first_ character of the current word if "within", else on
the first character of the preceding word (i.e., on the nearest beginning
of a word).

>   The same happens on Ctrl+RightArrow. All other editors places the
> cursor after the last letter of the following word, MSEide places the
> cursor at
>    the beginning of the next word, after the space between the two

Again, this editor does it differently (which I find annoying, BTW) - it
puts the cursor on the character _immediately behind_ the _last_ letter of
the current or follwing word.

And there are other ways to handle "word jumping".
BTW, I'd find your suggested bahaviour, "place the cursor before the first
letter of the current word", very annoying - I'm used to it being put _on_
the very first character of the next word.
On the other hand, for me, placing the cursor at the beginning of the next
word when jumping forward seems most "natural" to me, _not_ putting it
immediately behind its last letter.

> words.  Could this strange behaviour be a toggle option in the Editor

Well - isn't it so that the "strange behaviour" is the one _you_
suggest? ;->

There are so many ways an editor can be working that it's almost impossible
to implement all of them as options, I think.
Thus, forcedly, I took another approach to that - _I_ try to adjust to the
editor idiosyncrasies as much as possible, mostly by doing without some
well meant but annoyingly (for me) implemented features.

BTW, what editor do you refer to, actually? It cannot be any of the common
windows editors, including the Delphi IDE editor, since these all work
different from your proposal...
