On 16 August 2010 02:09, Sieghard wrote:
>
> An IDE is at least also something like a make program - it controls the
> build process, calling any needed programs to effect this, and gathers the
> output (and return codes) for evaluation by the user, _and_ attempts to
> provide information to the editor to allow for navigation by "compile
> problem".

jEdit, Syn Edit etc can do that as well, and they are called
"programmer editors" and not "IDE's". So from my pointt of view [they
way I use them], Lazarus IDE and MSEide are simply "enhanced
programmer editors" with better code navigation support, specific to
the Object Pascal language, and they know a bit more about the
compiler by introducing a dialog for compiler parameters (though
MSEide is generic enough to work with non-Pascal compilers too).

Anyway, not sure where we are going with this discussion. :)
