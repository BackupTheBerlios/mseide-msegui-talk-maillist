Hi Martin,

Since yesterday, I started getting this corrupt screen display when MSEide
is started. It actually happens at work and at home. The only way to fix
it, is to close the whole application and restart it. Normally after a
restart, then everything works fine again. Switching virtual desktops
doesn't help either, the vertical color bands just get more.

Work PC: 64-bit Ubuntu 10.04, FPC 2.4.3, Nvidia proprietary graphics driver
         Intel Quad core, 64-bit MSEide

Home PC: 32-bit Ubuntu 8.04.4, FPC 2.4.3, ATI proprietary graphics driver
         Intel P4, 32-bit MSEide


Both systems have MSEide r3836. And MSEide is compiled using the following
script.

----------------------
#!/bin/sh

rm units/*

fpc \
  -XX -Xs -ve -O1 -B \
  -FUunits \
  -Fi${PWD}/lib/common/kernel \
  -Fu${PWD}/lib/common/kernel -Fu${PWD}/lib/common/db \
  -Fu${PWD}/lib/common/designutils -Fu${PWD}/lib/common/dialogs \
  -Fu${PWD}/lib/common/editwidgets -Fu${PWD}/lib/common/i18n \
  -Fu${PWD}/lib/common/ifi -Fu${PWD}/lib/common/image \
  -Fu${PWD}/lib/common/opengl -Fu${PWD}/lib/common/pascalscript \
  -Fu${PWD}/lib/common/printer -Fu${PWD}/lib/common/regcomponents \
  -Fu${PWD}/lib/common/report -Fu${PWD}/lib/common/serialcomm \
  -Fu${PWD}/lib/common/sysutils -Fu${PWD}/lib/common/unicode \
  -Fu${PWD}/lib/common/widgets -Fu${PWD}/lib/common/kernel/i386-linux \
  -Fu${PWD}/lib/common/math \
  -dmse_withoutdb -dmse_no_ifi \
  -o./mseide.elf apps/ide/mseide.pas

----------------------
