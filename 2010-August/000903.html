Hi Martin,

This is a strange request. I believe fpGUI works correctly, and other
toolkits (including VCL) wrong, but I would like to follow the crowd.
:)

When I launch a MSEgui or VCL or LCL application (even just a single
form app, no buttons/labels etc) via the following code:

var
  p: TProcess;
begin
  p := TProcess.Create(self);
  p.CommandLine := FileNameedit1.Text;
  p.ShowWindow := swoHIDE;
  p.Execute;


... the main form of the application is always shown. Basically, the
app starts normally, as if you double clicked on it's icon. fpGUI
based apps on the other hand adhere to the swoHide, which means the
application starts up, but the main form is *not* shown. As you can
imagine, this does cause problems if the application only has one
form. :)

Unfortunately I can't take credit to adhering so well to the Windows
API rules - simply because I have no idea why it works. :-) Where in
MSEgui, or better yet, how do I ignore the swoHide (apparently Windows
STARTUPINFO record information) at application startup?

I debugged one of my apps (a single form application with no other
components), and traced the windows messages it receives. When I
double click the icon, it receives in order WM_NCActive,  WM_Active,
WM_Move, WM_Paint.  But when that same application is launched via
TProcess, I only receive WM_NCActive,  WM_Active.  The important
message WM_Paint is never received in the latter case.
