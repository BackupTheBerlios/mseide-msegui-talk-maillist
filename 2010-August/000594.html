The '!' denotes that gdb reports an error in the condition. On first stop on
the breakpoint there should be an errormessage like 'No symbol "name" in
current context.'. "FName" instead of "Name" solves that problem but the
next problem is that gdb can't handle Pascal ansi-string comparisons in
expressions. I often use a special test statement in order to stop for an
individual component:

"
if name = 'abcde' then begin
 testvar:= 0;    <<<<<<---- breakpoint
end;
"
All test statements use "testvar*" as variables so 'Search'-'Find in Files'
for "testvar" lists all test statements for deleting.

> Anyway, if Conditional Breakpoints are implemented, how am I supposed
> to type the Condition, so that the program only stops when the
> condition evaluates to True?
>
