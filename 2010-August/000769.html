Hallo Martin,

sorry - I'm here again with another itch...
Though this seems to be an FPC problem:

There was a strange effect when I tried to use the screen keypad - it
simply wouldn't display. Within my main project, it would even enter the
value given into the target variable, but within the test project, it
wouldn't even do that, just produce an empty value.

As within the "SetEntry" routine, I used a "WITH" statement to access the
entry field, I tried to qualify the accesses to this filed nonetheless - to
no avail, the result was still an empty string.
Then I tried to completely remove the WITH statement (it was only a matter
of commenting out the BEGIN and END lines any more) - and now, the keypad
was _fully_ _functional_!

Whether this is caused because this routine is inherited by the descendants
- although the entry field is so, too - I don't know. But the simple
measure of removing the WITH made it working.
Ther seems to be a real bug in FPC 2.4.0 concerning WITH statements -
I don't know whether it was already present before, as I didn't test these
things with the 2.2.x versions; the WITH staement _did_ work with
pre-version 2 compilers, and probabely with version 2.0.

Could someone perhaps check the issue whether it can be reproduced, so a
bug report could be issued? (Or does someone know whether a bug report has
been filed already on this?)
