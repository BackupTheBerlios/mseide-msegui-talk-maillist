A key press event will be "eaten" by the first function which sees the event 
and triggers an action. In your code 
"
 iscellclick(info,[ccr_dblclick])
"
returns true and sets the es_processed flag in the key event record so further 
processing of the key event will be stopped -> the return key event will not 
be delivered to the default button.
The functionality avoids the execution of several actions by a single 
keystroke.
