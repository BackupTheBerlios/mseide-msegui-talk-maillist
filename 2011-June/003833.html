> Seeing that nobody seems to have any real interest in starting, or at
> least actively developing a FPC based debugger, I took it upon myself
> to start such a project. I'm so sick of the bad debugger support via
> GDB, the huge delays between new FPC Dwarf support and when we can
> actually use it in a GDB release (or other way round), unable to debug
> properties, strings (under 64-bit), dynamic arrays etc...
>
> So I started reading up on how debuggers work - yes I know very little
> about debuggers. I'm also studying the duby and fpdebug projects for
> hints. The principle seems rather simple, it's more about reading the
> DWARF debug info to correctly map the assembler to Object Pascal code.
> Anyway, this is going to take a while to be fully working, but I am
> making in my mission to complete. :) I'll post notices in the FPC
> Users mailing list if I have something usable for others to try.
