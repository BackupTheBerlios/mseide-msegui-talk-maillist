On Wednesday 15 June 2011 10:19:59 Julio Jiménez wrote:
> Probably is as designed, but I think there is something wrong setting
> anchors.
>
> i.e.  I place a tlabel in a form. If I want the component to anchor
> right... I have to set an_left to false and an_right to true
> but there is a wrong behavior.
>
> When I set an_left to false, the widget is moved to the left side of the
> form.

and extended to the client area of the parent widget. tlabel is a special case 
because it has ow1_autowidth and ow1_autoheight set by default.

> To make it working ok you have first to set 
> an_right to true and then an_left to false. I think this is wrong. I have
> used other RAD and none of them have this behavior.
>
> From my point of view, setting an_left to true mustn't move the widget in
> the form.
>
> The same behavior you get setting an_top to false (widget is set in the top
> side of the form).
>
> Isn't this a bug?
>
No. Anchors without an_left,an_right and/or without an_top,an_bottom act the 
same as the Delphi setting alClient. So always first add the wanted anchors 
then remove the not wanted anchors.

Martin
