Hi,

I thought I asked this before, but I searched the newsgroup and couldn't
find any match.

What is the 'Project Tree' window actually for?  I use it like Lazarus's
"Project  Inspector" (Delphi  IDE had  something similar). I  use it  as
window that  holds a  reference to  all my  files (*.pas,  *.sql, *.ipf,
*.txt etc) that are part of my project.

Is this the intended usage?  Is  so, can it's content be stored/included
with the "export a  clean *.prj file - the "Save"  and "Load" buttons at
the bottom of the "Project Options" dialog.

I often work on multiple systems - using the same project. This would be
the case of multiple developers working on the same project too - a team
of  programmers. The project  file needs  to be  stored in  the revision
control  system, but  we don't  want each  developers personal  settings
(like  current  opened files,  editor  settings  etc) included  in  this
.prj  file.  Lazarus  does  this quite  nicely. It  calls such  personal
information as  "session information",  and it can  be defined  to store
those session info outside the project file. This is very handy.

MSEide's  "Project Options  > Save"  seems  to do  similar, but  doesn't
include  the "Project  Tree"  content. Thus my  question,  what is  that
dialog actually used for?
