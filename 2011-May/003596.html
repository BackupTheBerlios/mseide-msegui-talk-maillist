On Wednesday 11 May 2011 22:20:04 IvankoB for-mse wrote:
> Oops, it seems that me have remebered ( the Delphi experience ) what
> lookup fields are designed for - in-memory joining different datasets
> - for example, on key fields to show corresponding data values. Smth
> like a string grid bound to an SQL query to a single table but showing
> data from many tables - via lookuping to these external datasets.

Correct.

> Really, it has been the approach difficult to adopt.
> Which way is prefeble for DB grids - widgetgrid with tdb*editlb or
> stringgrid bound to query with field lookups ?
>
twidgetgrid is more versatile than tstringgid. For editable field values a 
dropdown edit (tdbenumedit*) is necessary, either the *lb or the *db version. 
lookupbuffers are faster than index based lookup fields but lookupbuffers 
must be reloaded if the data of the lookupdataset changes and the values are 
in memory twice if the dataset will not be closed after loading the 
lookupbuffer.
For display only field values lookup fields can be used either in a 
tdbstringgrid or a tdbwidgetgrid.
