> Try System.Assign(fileout, 'testfile.txt');

Yes, that's the solution using qualification taking advantage of the unit
structure. This is the most general solution, as it is always applicable.

And then there's the Turbo/Object Pascal solution for such name clashes
which provided alternate names for all - most - of the affected routines,
in case of file operations following the scheme <common name>file.
So here "AssignFile" would apply, and another candidate was "CloseFile",
as there might be a name clash with a "Close" object method also.
