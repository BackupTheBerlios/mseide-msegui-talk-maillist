The ansistring is used as bytestream in the choosen image format

> Lastly, if I want to manipulate my picture (eg. resizing) how do I then
> load the tmaskedbitmap into the tdataimage in the widgetgrid?
>
It must be converted to a supported image data format (JPG,PNG...)
BTW, the FPC image converting functions and the whole fpimage 
environment is not very fast, an impolite person even would say it is 
very slow. ;-)
Maybe it is better to use a tdbwidgetgrid, tdbdataimage and to store the 
image data in a tmsebufdataset descendant and to use a tmsegraphicfield. 
tmsegraphicfield has a cache for the converted images in displayable 
internal format. The size of the cache buffer can be set by 
tmsegraphicfield.imagecachekb (kilo bytes).
