MSEgui has "optional object properties", examples are twidget.frame and 
twidget.face. "optional object properties" are NIL by default and don't 
use more resources than the NIL instance pointer.
The property setter procedure creates the property object instance if 
the current object pointer is NIL.
The property getter function creates an object instance if the object 
pointer is NIL and the property owner component has csReading set in 
ComponentState.
Every optional object property has (should have) a accompanying create 
procedure, examples are twidget.createframe, twidget.createface. create* 
creates a property object instance if necessary.
An optional object property instance can be freed by setting the 
property to NIL.
An object property setter does not replace the existing instance pointer 
by the new value, instead it calls <oldvalue>.assign(<newvalue>).
Another kind of optional object properties are "parent object 
properties", for example tedit.font. Parent object properties return the 
instance pointer of the appropriate property of the parent object  if 
the own value is NIL.
So
"
<myobjectinstance>.theparentobjectproperty:=
                 <myobjectinstance>.theparentobjectproperty;
"
on a NIL property retrieves the property object instance of the parent 
object by the getter function, creates a new property object instance 
and calls <newobjectinstance>.assign(<parentobjectinstance>) by the 
setter procedure.
